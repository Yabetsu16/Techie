# Sprint 3 Retrospective

## What Went Well

- Successfully separated prompt engineering from application logic.
- Established Techie's teaching identity.
- Improved project architecture through `PromptService`.
- Added professional project tooling.
- Built a reusable evaluation framework.
- Evaluated Techie using structured teaching scenarios instead of subjective impressions.

---

## Challenges

### Over-explaining

Techie sometimes continued explaining after fully answering the learner's question.

---

### Advanced Concepts

Some responses introduced concepts beyond the learner's current level.

Examples:

- RAM
- Garbage Collection
- `range`
- implementation details

---

### Honest Uncertainty

The evaluation revealed that Techie occasionally speculates instead of explicitly acknowledging uncertainty.

This is the highest-priority improvement for a future sprint.

---

## Lessons Learned

Prompt engineering benefits from objective evaluation.

Instead of relying on intuition, structured evaluations revealed specific strengths and weaknesses that can guide future improvements.

---

## Action Items for Sprint 4

- Strengthen uncertainty handling.
- Reduce unnecessary elaboration.
- Encourage shorter introductions.
- Keep explanations focused on the learner's question.
- Expand the evaluation suite.
